## 아이디어 1. DailyDesk

**해결하는 문제**
소상공인은 하루에도 수십 번 매장/작업실과 이동하며 업무를 처리한다. 노트북을 열기 어려운 순간에도 "오늘 할 일, 재고 체크, 마감 확인"이 필요하다. 웹 툴은 꺼내기 불편해서 결국 머릿속에만 남고 누락이 발생한다.

**모바일이어야 하는 이유**
- 잠금화면/홈화면 위젯으로 오늘의 체크리스트를 항상 노출
- 아침 9시 "오늘 루틴 시작" 푸시, 마감 1시간 전 자동 알림
- 오프라인 상태(지하 창고, 매장 사각지대)에서도 작동

**핵심 기능**
- 요일별 루틴 템플릿 (매주 월: 정산, 화: SNS 등록 등)
- 홈 위젯에서 체크박스 직접 클릭
- 반복 미완료 항목 자동 감지 후 "이 항목 자주 빠집니다" 경고
- 마감일 기반 우선순위 자동 정렬
- 간단한 메모/사진 첨부 (영수증 촬영 후 태그)

**수익 모델**
- 무료: 루틴 3개, 위젯 기본형
- 프로 월 4,900원: 루틴 무제한, 위젯 커스텀, 알림 고급 설정

**예상 제작 기간**
- MVP: 6주 (Expo + React Native, 로컬 스토리지 우선)
- 위젯 연동: +2주 (expo-widgets or native module)

## 아이디어 2. SnapStock

**해결하는 문제**
핸드메이드 작가, 키링/굿즈 판매자, 소규모 제조업자는 재고를 엑셀로 관리하다 놓친다. 물건을 만들 때는 손에 흙/물감이 묻어 있고, 컴퓨터 앞에 앉을 여유가 없다. 결과적으로 "품절인데 주문받기", "재고 있는데 판매 중단" 실수가 반복된다.

**모바일이어야 하는 이유**
- 카메라로 제품 촬영 즉시 재고 등록 (웹에선 사진 업로드 흐름이 끊김)
- 재고 0개 도달 시 즉각 푸시 알림
- 작업대 옆에서 한 손으로 수량 +1/-1 탭

**핵심 기능**
- 카메라 촬영 즉시 상품 등록 (이름, 수량, 가격)
- QR/바코드 스캔으로 기존 상품 빠른 수량 조정
- 재고 임계값 설정 -> 알림 ("키링 재고 3개 남음")
- 판매 채널별 메모 (마플샵/스마트스토어/오프라인 구분)
- 월별 판매량 간단 그래프

**수익 모델**
- 무료: 상품 20개 이하
- 프로 월 5,900원: 상품 무제한, CSV 내보내기, 채널별 분리 관리

**예상 제작 기간**
- MVP: 5주 (카메라 연동 expo-camera, SQLite 로컬 DB)
- QR 스캔 추가: +1주

## 아이디어 3. CloseUp

**해결하는 문제**
1인 창업자는 잠재 고객과 DM, 오픈채팅, 명함 교환 등 다양한 경로로 접촉한다. 하지만 "저번에 관심 있다던 그분 어디 갔지?", "팔로업 언제 하기로 했더라"를 놓쳐서 거래 기회를 날린다. CRM은 너무 무겁고, 메모장은 검색이 안 된다.

**모바일이어야 하는 이유**
- 명함/QR 촬영 즉시 잠재고객 등록
- "X일 후 팔로업" 설정 -> 당일 아침 푸시 ("김OO님 팔로업 오늘입니다")
- 현장(전시회, 팝업)에서 바로 입력, 인터넷 없어도 저장

**핵심 기능**
- 연락처 + 메모 + 관심 상품 태그로 잠재고객 카드 생성
- 명함 카메라 촬영 -> OCR로 자동 이름/연락처 추출
- 팔로업 날짜 설정 + 푸시 리마인더
- 파이프라인 상태 관리 (접촉 - 관심 - 협의 - 성사 - 종료)
- 거래 성사 시 매출 메모 연동

**수익 모델**
- 무료: 잠재고객 30명 이하
- 프로 월 6,900원: 무제한, OCR 명함 스캔, CSV 백업

**예상 제작 기간**
- MVP: 7주 (파이프라인 UI + 푸시 알림이 핵심, expo-notifications)
- OCR 명함 스캔: +2주 (Google ML Kit 또는 외부 API)

## 추천 순서

1순위: SnapStock — 뀨스튜디오 키링 사업과 사장님 본인이 직접 타깃 유저. 자신의 고통을 해결하며 만들면 PMF 검증이 가장 빠름.

2순위: DailyDesk — 타깃이 넓고 위젯/루틴 수요는 꾸준함. 기존 RetroBot, OpsPack 기획과 연계 가능.

3순위: CloseUp — 잠재력은 가장 크지만 OCR·파이프라인 UI로 개발 볼륨이 상대적으로 큼.
