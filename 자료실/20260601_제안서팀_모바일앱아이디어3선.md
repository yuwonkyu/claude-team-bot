사장님은 Next.js, Claude AI, React Native/Expo 스택을 보유한 1인 개발자이며,

## 아이디어 1: instalink Mobile

**앱명:** instalink (모바일 앱 확장)

**한 줄 핵심 가치:**
"인스타 링크인바이오, 이제 앱에서 바로 편집하고 통계까지 확인한다"

**타깃:**
- 기존 instalink 웹 유료 구독자 (즉시 설치 가능한 기존 고객 존재)
- 인스타그램 마켓터, 소상공인, 1인 크리에이터
- 외출 중에도 링크 수정·통계 확인이 필요한 사용자

**수익 모델:**
- 웹 구독자에게 앱은 무료 번들 제공 (구독 유지 유인)
- 앱 단독 구독: 월 3,900원 / 연 29,000원 (인앱 결제)
- 프리미엄 기능 (AI 링크 추천, 클릭 히트맵) 은 상위 플랜에만 제공

**기존 자산 연계:**
- instalink 백엔드 API를 그대로 사용. 신규 개발 부담 최소화
- 웹과 앱이 동일 계정 연동 → 유저 전환 비용 0
- 토스페이먼츠 구독 결제 인프라는 웹에서 이미 심사 중 → 앱 인앱 결제와 병행 운영

**예상 MVP 기간:** 4~5주
- 1주: 기존 API 정리 및 앱 라우팅 설계
- 2주: 링크 편집 / 통계 뷰 핵심 화면 개발 (Expo)
- 1주: 로그인, 구독 상태 연동 테스트
- 1주: 앱스토어 심사 제출 및 대기

**왜 이게 먼저인가:**
기존 유저가 있고, API가 있고, 브랜드가 있다.
신규 사업이 아니라 기존 자산의 수익 채널 확장이다.
앱스토어 등록만으로 "instalink는 앱도 있다"는 신뢰도가 높아진다.

## 아이디어 2: DailyLink — 하루 1링크 뉴스레터 앱

**앱명:** DailyLink

**한 줄 핵심 가치:**
"매일 오전 8시, 오늘 꼭 봐야 할 링크 하나만 푸시로 보내준다"

**타깃:**
- 정보 과부하에 지친 20~35세 직장인, 프리랜서
- 뉴스레터는 구독하지만 메일함을 잘 안 여는 사람
- 하루에 링크 하나만 소비하는 '미니멀 큐레이션' 선호층

**수익 모델:**
- 무료: 하루 1링크 (카테고리 선택 가능)
- 프리미엄 월 2,900원: 카테고리별 5링크, 북마크, 아카이브 검색
- 스폰서십: 뉴스레터 발행자가 "오늘의 링크" 슬롯 구매 (CPM 광고)
- 큐레이터 파트너십: instalink 사용자가 DailyLink에 링크 기고 → 양쪽 플랫폼 시너지

**기존 자산 연계:**
- instalink 링크 데이터 및 클릭 통계를 큐레이션 알고리즘에 활용
- kkustudio.com 사장님 개인 브랜드로 초기 구독자 유입 (팬 기반 활성화)
- instalink 유저들이 자신의 링크를 DailyLink에 제출할 수 있는 연동 흐름 설계 가능

**예상 MVP 기간:** 5~6주
- 1~2주: 큐레이션 CMS 설계 (Notion + Claude AI로 초안 자동화 가능)
- 2~3주: 앱 UI (홈 피드, 푸시 설정, 북마크) Expo 개발
- 1주: 푸시 알림 인프라 (Expo Notifications) 구성 및 테스트
- 1주: 앱스토어 심사

**왜 이게 유망한가:**
큐레이션 콘텐츠 앱은 운영 자동화가 가능하다.
Claude AI로 링크 요약·태깅을 자동화하면 1인이 매일 발행해도 부담이 없다.
광고 슬롯 수익은 초기부터 현금화 가능하고,
구독자가 쌓이면 뉴스레터 발행자들이 먼저 제휴를 요청해온다.

## 아이디어 3: KkuKit Mobile — 개발자 도구 모음 앱

**앱명:** KkuKit (모바일 버전)

**한 줄 핵심 가치:**
"개발자가 이동 중에도 쓰는 유틸 도구 모음 — 색상 추출, JSON 포맷터, base64 변환 등"

**타깃:**
- 프리랜서 개발자, 학생 개발자, 퍼블리셔
- 노트북 없이 이동 중 간단한 변환/확인 작업이 필요한 사람
- "이 도구 하나만 있으면 되는데" 를 자주 느끼는 개발자

**수익 모델:**
- 무료: 기본 도구 10개 (색상 추출, JSON, base64, URL 인코딩 등)
- 프리미엄 일회성 결제 4,900원: 도구 전체 잠금 해제 (Lifetime 판매)
- 팀 플랜 월 4,900원: 도구 세팅 공유, 히스토리 동기화
- 앱스토어 피처링 노릴 수 있는 카테고리 (Utilities, Developer Tools)

**기존 자산 연계:**
- KkuKit 웹 기획이 이미 존재 → 웹에서 검증된 도구 목록을 앱으로 이식
- kkustudio.com에서 앱 소개 랜딩으로 연결 → 유입 경로 확보
- 개발자 커뮤니티(인디해커, 노마드코더 등) 홍보 시 사장님 브랜드와 연결해 신뢰도 확보

**예상 MVP 기간:** 3~4주
- 1주: 도구 10개 선정 및 로직 구현 (대부분 순수 JS 함수)
- 1~2주: Expo 앱 UI 구성 (탭 네비게이션, 각 도구 화면)
- 1주: 인앱 결제 (RevenueCat 연동) 및 앱스토어 심사 준비

**왜 이게 실용적인가:**
유틸 앱은 서버 비용이 거의 없다. 오프라인에서도 동작한다.
한 번 만들면 거의 수정 없이 수익이 발생하는 구조다.
일회성 결제(Lifetime) 모델은 "구독 피로" 없이 구매 전환율이 높다.
KkuKit 웹 기획이 이미 있으므로 설계 단계를 건너뛸 수 있다.

## 3가지 비교 요약

| 항목 | instalink Mobile | DailyLink | KkuKit Mobile |
|------|-----------------|-----------|---------------|
| MVP 기간 | 4~5주 | 5~6주 | 3~4주 |
| 초기 유저 확보 | 기존 instalink 구독자 즉시 전환 | 사장님 팬베이스 + SNS | 개발자 커뮤니티 |
| 서버 비용 | 기존 인프라 활용 | 낮음 (자동화 가능) | 거의 없음 |
| 수익 구조 | 월 구독 | 월 구독 + 광고 | 일회성 결제 |
| 기존 자산 연계 | 매우 높음 | 중간 | 높음 |
| 운영 난이도 | 낮음 | 중간 (큐레이션 필요) | 매우 낮음 |
| 성장 잠재력 | 중간 (기존 시장 확장) | 높음 (콘텐츠 복리 효과) | 중간 (유틸 특성상 상한 존재) |

## 제안서팀 권고

**우선순위 1: instalink Mobile**
리스크가 가장 낮고, 기존 구독자를 앱 채널로 전환하는 즉효 전략이다.
토스페이먼츠 심사 완료(6/20 예정) 이후 앱 인앱 결제와 병행하면 수익 채널이 이중화된다.

**우선순위 2: KkuKit Mobile**
개발 기간이 가장 짧고 서버 비용이 없다. instalink 개발 병행 또는 완료 후 빠르게 착수 가능.
"개발자 유틸 앱"은 앱스토어 자연 유입이 꾸준하다.

**우선순위 3: DailyLink**
성장 잠재력은 가장 높지만 큐레이션 운영 루틴이 필요하다.
Claude AI 자동화를 먼저 구성한 뒤 착수하는 것을 권장한다.
